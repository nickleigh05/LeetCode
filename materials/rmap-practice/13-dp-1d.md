### 70. Climbing Stairs — Easy
[LeetCode](https://leetcode.com/problems/climbing-stairs/)  
[Solution file (no hints)](../../problems/0001-0499/70.py)

Count the ways to climb n stairs taking 1 or 2 steps at a time. Why does the number of ways to reach step n equal the sum of ways to reach step n-1 and step n-2?

<details>
<summary>Hint</summary>

This is Fibonacci in disguise (see [Dynamic Programming](../algorithms/dynamic-programming.md)): the last move to reach step n was either a 1-step from n-1, or a 2-step from n-2, so `ways(n) = ways(n-1) + ways(n-2)`.
</details>

<details>
<summary>Solution</summary>

```python
if n <= 2:
    return n

one_back, two_back = 2, 1              # ways(n-1), ways(n-2)
for _ in range(3, n + 1):                 # for loop building up from step 3
    one_back, two_back = one_back + two_back, one_back

return one_back
```

Building blocks: [if-return](../syntax/if-return.md) · [for-loop](../syntax/for-loop.md) · [swap-tuple-assign](../syntax/swap-tuple-assign.md)
</details>

<details>
<summary>Time & space complexity</summary>

**Time: O(n)** — one pass building up from the base cases.
**Space: O(1)** — only two running variables (space-optimized from an O(n) DP array).
</details>

### 746. Min Cost Climbing Stairs — Easy
[LeetCode](https://leetcode.com/problems/min-cost-climbing-stairs/)  
[Solution file (no hints)](../../problems/0500-0999/746.py)

Given a cost at each step, find the minimum cost to reach the top (starting from step 0 or 1). Why does the cheapest way to reach step i only depend on the cheapest ways to reach steps i-1 and i-2?

<details>
<summary>Hint</summary>

[DP](../algorithms/dynamic-programming.md): `cost_to_reach(i) = cost[i] + min(cost_to_reach(i-1), cost_to_reach(i-2))`. Build this bottom-up, then the answer is the min of the two ways to step past the last stair.
</details>

<details>
<summary>Solution</summary>

```python
n = len(cost)
one_back, two_back = 0, 0              # cost to reach step n-1, step n-2

for i in range(2, n + 1):                # for loop building up from the ground
    current = min(one_back + cost[i - 1], two_back + cost[i - 2])
    two_back, one_back = one_back, current

return one_back
```

Building blocks: [for-loop](../syntax/for-loop.md) · [comparison-operators](../syntax/comparison-operators.md) (`min()`) · [swap-tuple-assign](../syntax/swap-tuple-assign.md)
</details>

<details>
<summary>Time & space complexity</summary>

**Time: O(n)** — one pass over the steps.
**Space: O(1)** — only two running variables.
</details>

### 198. House Robber — Medium
[LeetCode](https://leetcode.com/problems/house-robber/)  
[Solution file (no hints)](../../problems/0001-0499/198.py)

Rob houses in a line, maximizing loot, without robbing two adjacent houses. At each house, what's the choice between "rob it" and "skip it," and what do you need to know to make that choice?

<details>
<summary>Hint</summary>

[DP](../algorithms/dynamic-programming.md): at each house, either skip it (carry forward the best so far) or rob it (this house's value plus the best from two houses back, since the adjacent one is off-limits).
</details>

<details>
<summary>Solution</summary>

```python
rob1, rob2 = 0, 0                      # best up to i-2, best up to i-1

for num in nums:                         # for loop over each house
    new_rob = max(rob1 + num, rob2)         # rob this house, or skip it
    rob1 = rob2
    rob2 = new_rob

return rob2
```

Building blocks: [for-loop](../syntax/for-loop.md) · [comparison-operators](../syntax/comparison-operators.md) (`max()`)
</details>

<details>
<summary>Time & space complexity</summary>

**Time: O(n)** — one pass over the houses.
**Space: O(1)** — only two running variables.
</details>

### 213. House Robber II — Medium
[LeetCode](https://leetcode.com/problems/house-robber-ii/)  
Solution: not yet solved in this repo.

Same as [198](#198-house-robber--medium), but houses are arranged in a circle (first and last are adjacent). Why does running the linear solution twice — once excluding the first house, once excluding the last — handle the wraparound?

<details>
<summary>Hint</summary>

Since the first and last houses can't both be robbed, run the [198](#198-house-robber--medium) [DP](../algorithms/dynamic-programming.md) twice: once on `houses[:-1]` (excluding the last), once on `houses[1:]` (excluding the first), and take the max of the two results.
</details>

<details>
<summary>Solution</summary>

```python
def rob_linear(houses):
    rob1, rob2 = 0, 0
    for num in houses:
        rob1, rob2 = rob2, max(rob1 + num, rob2)
    return rob2

if len(nums) == 1:
    return nums[0]

return max(rob_linear(nums[:-1]), rob_linear(nums[1:]))   # exclude last, exclude first
```

Building blocks: [list-slicing](../syntax/list-slicing.md) · [for-loop](../syntax/for-loop.md) · [comparison-operators](../syntax/comparison-operators.md) (`max()`)
</details>

<details>
<summary>Time & space complexity</summary>

**Time: O(n)** — two linear passes over the houses.
**Space: O(1)** — only a couple of running variables per pass.
</details>

### 5. Longest Palindromic Substring — Medium
[LeetCode](https://leetcode.com/problems/longest-palindromic-substring/)  
Solution: not yet solved in this repo.

Find the longest palindromic substring. Why does expanding outward from each possible *center* (single character or between two characters) cover every possible palindrome?

<details>
<summary>Hint</summary>

For each index, expand outward with two pointers while characters match, once treating the index as an odd-length center and once as an even-length center (pairing it with the next index). Track the longest expansion seen.
</details>

<details>
<summary>Solution</summary>

```python
def expand(l, r):
    while l >= 0 and r < len(s) and s[l] == s[r]:   # expand while it stays a palindrome
        l -= 1
        r += 1
    return s[l + 1:r]                                  # shrink back to the last valid match

best = ""
for i in range(len(s)):                    # for loop over every possible center
    odd = expand(i, i)                        # odd-length palindrome centered at i
    even = expand(i, i + 1)                   # even-length palindrome centered between i, i+1
    best = max(best, odd, even, key=len)

return best
```

Building blocks: [while-loop](../syntax/while-loop.md) · [for-loop](../syntax/for-loop.md) · [sorting-key](../syntax/sorting-key.md) (`max(..., key=len)`)
</details>

<details>
<summary>Time & space complexity</summary>

**Time: O(n²)** — n centers, each expansion up to O(n).
**Space: O(1)** extra beyond the returned substring.
</details>

### 647. Palindromic Substrings — Medium
[LeetCode](https://leetcode.com/problems/palindromic-substrings/)  
Solution: not yet solved in this repo.

Count how many substrings of a string are palindromes. How does the same center-expansion idea from [5](#5-longest-palindromic-substring--medium) let you *count* instead of just find the longest?

<details>
<summary>Hint</summary>

Expand from every center (odd and even, same as [5](#5-longest-palindromic-substring--medium)), and count one palindrome for every successful expansion step rather than tracking the longest.
</details>

<details>
<summary>Solution</summary>

```python
def count_expand(l, r):
    count = 0
    while l >= 0 and r < len(s) and s[l] == s[r]:   # every successful expansion is a palindrome
        count += 1
        l -= 1
        r += 1
    return count

total = 0
for i in range(len(s)):                    # for loop over every possible center
    total += count_expand(i, i)               # odd-length centers
    total += count_expand(i, i + 1)           # even-length centers

return total
```

Building blocks: [while-loop](../syntax/while-loop.md) · [for-loop](../syntax/for-loop.md) · [arithmetic-operators](../syntax/arithmetic-operators.md) (`+=`)
</details>

<details>
<summary>Time & space complexity</summary>

**Time: O(n²)** — n centers, each expansion up to O(n).
**Space: O(1)** — only a running counter.
</details>

### 91. Decode Ways — Medium
[LeetCode](https://leetcode.com/problems/decode-ways/)  
Solution: not yet solved in this repo.

A digit string can decode to letters (1='A' ... 26='Z'); count the number of ways to decode it. At each position, when can you decode just one digit, and when can you also decode it paired with the previous digit?

<details>
<summary>Hint</summary>

[DP](../algorithms/dynamic-programming.md) from the end backward: `ways(i) = ways(i+1)` if `s[i] != "0"`, plus `ways(i+2)` if the two-digit number `s[i:i+2]` is between 10 and 26.
</details>

<details>
<summary>Solution</summary>

```python
n = len(s)
dp_next, dp_next2 = 1, 0               # ways(n) = 1 (empty string), ways(n+1) = 0 (unused base)

for i in range(n - 1, -1, -1):            # for loop building backward from the end
    if s[i] == "0":                         # a leading zero can never decode alone
        current = 0
    else:
        current = dp_next
        if i + 1 < n and 10 <= int(s[i:i + 2]) <= 26:  # valid two-digit decode
            current += dp_next2
    dp_next2 = dp_next
    dp_next = current

return dp_next
```

Building blocks: [range-function](../syntax/range-function.md) (reverse step) · [chained-comparisons](../syntax/chained-comparisons.md) · [type-conversion](../syntax/type-conversion.md) (`int()`)
</details>

<details>
<summary>Time & space complexity</summary>

**Time: O(n)** — one backward pass over the digits.
**Space: O(1)** — only two running variables.
</details>

### 322. Coin Change — Medium
[LeetCode](https://leetcode.com/problems/coin-change/)  
[Solution file (no hints)](../../problems/0001-0499/322.py)

Find the fewest coins needed to make up an amount (or -1 if impossible). Why does building up the answer for every amount from 0 to target, using already-solved smaller amounts, beat trying every combination of coins directly?

<details>
<summary>Hint</summary>

Bottom-up [DP](../algorithms/dynamic-programming.md): `min_coins(amount) = 1 + min(min_coins(amount - coin) for every coin)`. Build an array indexed by amount, starting from 0.
</details>

<details>
<summary>Solution</summary>

```python
dp = [float("inf")] * (amount + 1)
dp[0] = 0                                # 0 coins needed to make amount 0

for a in range(1, amount + 1):             # for loop building up every amount
    for coin in coins:                       # try using each coin as the last one added
        if a - coin >= 0:
            dp[a] = min(dp[a], 1 + dp[a - coin])

return dp[amount] if dp[amount] != float("inf") else -1
```

Building blocks: [list-basics](../syntax/list-basics.md) · [for-loop](../syntax/for-loop.md) (nested) · [comparison-operators](../syntax/comparison-operators.md) (`min()`)
</details>

<details>
<summary>Time & space complexity</summary>

**Time: O(amount · len(coins))** — for every amount, try every coin.
**Space: O(amount)** — the DP array.
</details>

### 152. Maximum Product Subarray — Medium
[LeetCode](https://leetcode.com/problems/maximum-product-subarray/)  
Solution: not yet solved in this repo.

Find the contiguous subarray with the largest product. Why must you track both the running max *and* running min product, unlike [Kadane's algorithm](../algorithms/kadane-algorithm.md) for sums?

<details>
<summary>Hint</summary>

A negative number can turn the smallest (most negative) running product into the largest one. Track both `cur_max` and `cur_min` at each step, since either could become the new max after multiplying by a negative number.
</details>

<details>
<summary>Solution</summary>

```python
res = max(nums)
cur_max, cur_min = 1, 1

for num in nums:                         # for loop, one pass
    if num == 0:                            # reset: a zero breaks any subarray
        cur_max, cur_min = 1, 1
        continue
    tmp = cur_max * num
    cur_max = max(num, tmp, cur_min * num)    # a negative num could make cur_min * num the biggest
    cur_min = min(num, tmp, cur_min * num)
    res = max(res, cur_max)

return res
```

Building blocks: [for-loop](../syntax/for-loop.md) · [break-continue](../syntax/break-continue.md) · [comparison-operators](../syntax/comparison-operators.md) (`max()`, `min()`)
</details>

<details>
<summary>Time & space complexity</summary>

**Time: O(n)** — one pass over the array.
**Space: O(1)** — a few running variables.
</details>

### 139. Word Break — Medium
[LeetCode](https://leetcode.com/problems/word-break/)  
Solution: not yet solved in this repo.

Determine if a string can be segmented into words from a dictionary. Why does "can `s[i:]` be segmented" only depend on trying every possible first word and checking if the *rest* can also be segmented?

<details>
<summary>Hint</summary>

[DP](../algorithms/dynamic-programming.md) from the end backward: `can_break(i)` is True if some `word` in the dictionary matches `s[i:i+len(word)]` *and* `can_break(i + len(word))` is also True.
</details>

<details>
<summary>Solution</summary>

```python
word_set = set(wordDict)
n = len(s)
dp = [False] * (n + 1)
dp[n] = True                             # base case: empty remainder is always breakable

for i in range(n - 1, -1, -1):             # for loop building backward from the end
    for w in word_set:                       # try every word as the next piece
        if i + len(w) <= n and s[i:i + len(w)] == w and dp[i + len(w)]:
            dp[i] = True
            break

return dp[0]
```

Building blocks: [set-basics](../syntax/set-basics.md) · [list-basics](../syntax/list-basics.md) · [range-function](../syntax/range-function.md) (reverse step) · [break-continue](../syntax/break-continue.md)
</details>

<details>
<summary>Time & space complexity</summary>

**Time: O(n² · m)** — n string positions, each trying up to m dictionary words with O(n) slicing/comparison.
**Space: O(n + total dictionary size)** — the DP array and the word set.
</details>

### 300. Longest Increasing Subsequence — Medium
[LeetCode](https://leetcode.com/problems/longest-increasing-subsequence/)  
Solution: not yet solved in this repo.

Find the length of the longest strictly increasing subsequence. Why does maintaining a "tails" array — the smallest possible tail value for an increasing subsequence of each length — let binary search replace an O(n²) comparison?

<details>
<summary>Hint</summary>

Keep an array `tails` where `tails[k]` is the smallest tail value seen for a length-`k+1` increasing subsequence. For each number, [binary search](../algorithms/binary-search.md) `tails` for where it belongs, replacing that spot (or appending if it extends the longest sequence).
</details>

<details>
<summary>Solution</summary>

```python
import bisect

tails = []
for num in nums:                          # for loop over the array
    pos = bisect.bisect_left(tails, num)     # where num would be inserted to keep tails sorted
    if pos == len(tails):                     # num extends the longest sequence so far
        tails.append(num)
    else:                                     # num replaces a tail, keeping options open
        tails[pos] = num

return len(tails)
```

Building blocks: [for-loop](../syntax/for-loop.md) · [list-basics](../syntax/list-basics.md) · [if-return](../syntax/if-return.md) (bisect module for binary search)
</details>

<details>
<summary>Time & space complexity</summary>

**Time: O(n log n)** — n numbers, each doing a binary search over `tails`.
**Space: O(n)** — the `tails` array, up to length n.
</details>

### 416. Partition Equal Subset Sum — Medium
[LeetCode](https://leetcode.com/problems/partition-equal-subset-sum/)  
Solution: not yet solved in this repo.

Determine if an array can be split into two subsets with equal sums. Why does this reduce to "does some subset sum to exactly half the total" — a classic 0/1 knapsack shape?

<details>
<summary>Hint</summary>

If the total sum is odd, it's impossible. Otherwise this is 0/1 knapsack (see [Dynamic Programming](../algorithms/dynamic-programming.md)): track which sums are achievable using a [hashset](../data-structures/hashset.md), adding each number to every currently-achievable sum.
</details>

<details>
<summary>Solution</summary>

```python
total = sum(nums)
if total % 2:                            # odd total can't split evenly
    return False

target = total // 2
achievable = {0}                          # sums reachable using numbers seen so far

for num in nums:                           # for loop over each number
    new_sums = {s + num for s in achievable if s + num <= target}   # extend existing sums
    achievable |= new_sums

return target in achievable
```

Building blocks: [set-comprehension](../syntax/set-comprehension.md) · [for-loop](../syntax/for-loop.md) · [set-operations](../syntax/set-operations.md) (`|=`)
</details>

<details>
<summary>Time & space complexity</summary>

**Time: O(n · target)** — n numbers, each potentially extending up to `target` achievable sums.
**Space: O(target)** — the achievable-sums set is bounded by the target value.
</details>
